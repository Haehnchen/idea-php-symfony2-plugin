Detects invalid {% from _self import ... %} usage inside Symfony UX Twig Components

Inside a Twig component context (<twig:Component> or {% component 'Name' %}), _self does not refer to the current template. Macro imports via _self will silently fail at runtime. Use the full template path instead.

Invalid:


{% macro message_formatter(message) %}
    <strong>{{ message }}</strong>
{% endmacro %}

<twig:Alert>
    {% from _self import message_formatter %}
    {{ message_formatter('Hello') }}
</twig:Alert>

Valid:


{% macro message_formatter(message) %}
    <strong>{{ message }}</strong>
{% endmacro %}

<twig:Alert>
    {% from 'path/to/this/template.html.twig' import message_formatter %}
    {{ message_formatter('Hello') }}
</twig:Alert>
