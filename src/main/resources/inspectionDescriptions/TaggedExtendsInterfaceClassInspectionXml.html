Service class needs to implement required interface for its tag

This inspection detects Symfony services that are tagged with interfaces but don't implement the required interface. In Symfony's dependency injection container, certain tags require the service class to implement specific interfaces to function correctly.

The inspection checks for:

  • Services with tags that require specific interfaces
  • Service classes that don't implement the required interface for their tag
  • Tagged services that would fail at runtime due to missing interface implementation

Example XML service configuration that would trigger this inspection: 


<services>
    <!-- This will trigger inspection - EventSubscriberInterface is required -->
    <service id="my_subscriber" class="App\Service\MySubscriber">
        <tag name="kernel.event_subscriber" />
    </service>

    <!-- This will trigger inspection - CommandInterface is required -->
    <service id="my_command" class="App\Service\MyTask">
        <tag name="console.command" />
    </service>

    <!-- This is valid - implements required interface -->
    <service id="valid_subscriber" class="App\Event\MyEventSubscriber">
        <tag name="kernel.event_subscriber" />
    </service>
</services>
