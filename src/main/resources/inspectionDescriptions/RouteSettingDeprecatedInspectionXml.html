Detects deprecated Symfony route configuration settings in XML routing files.

This inspection analyzes XML routing files and identifies deprecated route configuration options that should be replaced with their modern equivalents in newer Symfony versions.

The inspection checks for deprecated settings in:

  • pattern attribute - Should be replaced with path attribute
  • _method requirement - Should use HTTP method constraints directly
  • _scheme requirement - Should use scheme constraints directly

Example deprecated XML configuration: 



<routes>
    <route id="blog_show"
           controller="App\Controller\BlogController::show"
           pattern="/blog/{slug}">  <!-- DEPRECATED: 'pattern' should be 'path' -->
        <requirement key="_method">GET</requirement>  <!-- DEPRECATED -->
        <requirement key="_scheme">https</requirement> <!-- DEPRECATED -->
    </route>
</routes>

Example modern XML configuration: 



<routes>
    <route id="blog_show"
           controller="App\Controller\BlogController::show"
           path="/blog/{slug}">  <!-- MODERN: Use 'path' instead of 'pattern' -->
        <!-- MODERN: Use 'methods' and 'schemes' directly -->
        <default key="_controller">App\Controller\BlogController::show</default>
    </route>

    <!-- Alternative with method and scheme constraints -->
    <route id="blog_show_secure"
           path="/blog/{slug}"
           controller="App\Controller\BlogController::show"
           methods="GET"
           schemes="https" />
</routes>
