Detects invalid class references in Twig enum() and enum_cases() functions

This inspection validates that class names provided to enum() and enum_cases() Twig functions:

  • Exist in the project
  • Are actually PHP enum types (not regular classes)

The inspection checks for issues in:

  • {{ enum('App\\Config\\SomeOption') }}
  • {{ enum_cases('App\\Config\\SomeOption') }}
  • {% if enum('App\\Config\\SomeOption') == 'value' %}
  • {% set options = enum_cases('App\\Config\\SomeOption') %}

Example Twig template: 


{# Valid: enum class exists and is an enum type #}
{{ enum('App\\Enum\\Status') }}

{# Error: class doesn't exist #}
{{ enum('App\\Enum\\MissingEnum') }}

{# Warning: class exists but is not an enum #}
{{ enum('App\\Entity\\User') }}

{# Valid: returns all cases of the enum #}
{% for status in enum_cases('App\\Enum\\Status') %}
    {{ status.value }}
{% endfor %}

When an issue is detected:

  • Missing class: The specified class cannot be found in the project
  • Not an enum: The class exists but is not an enum type
