Constant referenced in XML configuration is not defined or cannot be resolved

This inspection detects undefined constants in Symfony XML configuration files. It validates that constants referenced using <argument type="constant"> are actually defined and available.

The inspection checks for:

  • Undefined PHP constants in <argument type="constant"> elements
  • Constants that don't exist in the current PHP context
  • Misspelled constant names

Example XML configuration that would trigger this inspection: 


<services>
    <service id="my_service" class="App\Service\MyService">
        <!-- Valid constant references -->
        <argument type="constant">DateTime::class</argument>
        <argument type="constant">PHP_VERSION</argument>

        <!-- Invalid constant that will trigger inspection -->
        <argument type="constant">\Foobar\Car::UNDEFINED_CONSTANT</argument>

        <!-- If MAX_ITEMS doesn't exist on MyService class, this will trigger inspection -->
        <argument type="constant">App\Service\MyService::MAX_ITEMS</argument>
    </service>

    <service id="another_service" class="DateTime">
        <argument type="constant">\Foobar\Car::FOOBAR_1</argument> <!-- This will trigger inspection -->
    </service>
</services>
