Detects references to Symfony routes in PHP code that cannot be found in routing configuration.

This inspection analyzes PHP code and identifies when route names used in Symfony URL generation methods don't correspond to any routes defined in the Symfony routing configuration.

The inspection checks for missing routes in:

  • $this->generateUrl('route_name')
  • $this->redirectToRoute('route_name')
  • $urlGenerator->generate('route_name')
  • $this->forward('App\\Controller\\SomeController::method')

Example PHP controller: 


class BlogController extends AbstractController
{
    public function redirectToHome(): Response
    {
        // This will be highlighted if 'homepage' route doesn't exist
        return $this->redirectToRoute('homepage');
    }

    public function showProduct(int $id): Response
    {
        // This will be highlighted if 'product_detail' route doesn't exist
        $url = $this->generateUrl('product_detail', ['id' => $id]);
        return new RedirectResponse($url);
    }

    public function redirectToAdmin(): Response
    {
        // This will be highlighted if 'admin_dashboard' route doesn't exist
        return $this->redirect($this->generateUrl('admin_dashboard'));
    }
}

Example PHP service with UrlGenerator: 


class NotificationService
{
    private $urlGenerator;

    public function __construct(UrlGeneratorInterface $urlGenerator)
    {
        $this->urlGenerator = $urlGenerator;
    }

    public function sendEmailNotification(User $user): void
    {
        // This will be highlighted if 'user_profile' route doesn't exist
        $profileUrl = $this->urlGenerator->generate('user_profile', [
            'id' => $user->getId()
        ]);
    }
}

When a missing route is detected, the inspection provides quick fixes:

  • Suggest possible typo corrections in the route name
  • Help identify the correct route name from available routes
