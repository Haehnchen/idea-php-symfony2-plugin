Detects mixing of Twig block syntax and HTML component syntax in Symfony UX Twig Components

When using Symfony UX Twig Components, you cannot mix the two component syntaxes. If the outer component uses HTML syntax (<twig:ComponentName>), inner blocks must also use HTML syntax (<twig:block name="...">), not Twig block tags.

Invalid (mixed syntax):


<twig:Card>
    {% block footer %}
        content
    {% endblock %}
</twig:Card>

Valid (consistent HTML syntax):


<twig:Card>
    <twig:block name="footer">
        content
    </twig:block>
</twig:Card>

Valid (consistent Twig syntax):


{% component 'Card' %}
    {% block footer %}
        content
    {% endblock %}
{% endcomponent %}
