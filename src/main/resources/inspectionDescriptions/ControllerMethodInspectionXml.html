Controller action method referenced in route does not exist in the specified class

This inspection detects missing controller action methods in Symfony XML route definitions. It validates that when a route references a controller action, the corresponding method actually exists in the controller class.

The inspection checks for:

  • Controller methods referenced in XML route configurations that don't exist
  • Misspelled method names in controller references
  • Controllers that exist but lack the specified action method

Example XML route configuration that would trigger this inspection: 


<routes>
    <route id="homepage" controller="App\Controller\HomeController::index" />
        <!-- OK if index() method exists in HomeController -->

    <route id="contact" controller="App\Controller\ContactController::show" />
        <!-- Triggers inspection if show() method doesn't exist in ContactController -->

    <route id="product_list" controller="App\Controller\ProductController::listProducts" />
        <!-- Triggers inspection if listProducts() method doesn't exist -->
</routes>

The inspection supports various controller reference formats:

  • App\Controller\MyController::methodName - Class::method syntax
  • MyController::methodName - Shortcut syntax
  • service_id:methodName - Service ID syntax
