Detects when a Symfony Command __invoke() method returns a non-integer value

Symfony 7.3+ invokable commands using the #[AsCommand] attribute must return an integer exit code from the __invoke() method.

Valid return values are:

  • Command::SUCCESS (0) - Command completed successfully
  • Command::FAILURE (1) - Command failed
  • Command::INVALID (2) - Invalid input

Returning non-integer values (strings, null, objects) will cause a TypeError at runtime.

See Symfony Console documentation for more information.
