Service argument is missing or does not match constructor signature

This inspection detects missing arguments in Symfony service definitions within YAML configuration files. It analyzes service definitions to identify when required constructor or method arguments are not properly configured.

The inspection checks for:

  • Missing constructor arguments in service definitions
  • Arguments that don't match the service class constructor signature
  • Required parameters that are not provided in the YAML configuration
  • Services that are not using autowiring and have missing arguments

The inspection ignores services with:

  • parent key (child services)
  • factory keys (factory-based services)
  • abstract: true
  • autowire: true (autowired services)

Example YAML service definition that would trigger this inspection: 


services:
    my_service:
        class: App\Service\MyService
        # Missing arguments section

    another_service:
        class: App\Service\AnotherService
        arguments:
            - '@doctrine.orm.entity_manager'
            # Missing second required argument

    # This service would NOT trigger inspection (autowired)
    autowired_service:
        class: App\Service\AutowiredService
        autowire: true
