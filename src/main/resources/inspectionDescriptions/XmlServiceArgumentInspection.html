Service argument is missing or does not match constructor signature

This inspection detects missing arguments in Symfony service definitions within XML configuration files. It analyzes service definitions to identify when required constructor or method arguments are not properly configured.

The inspection checks for:

  • Missing constructor arguments in service definitions
  • Arguments that don't match the service class constructor signature
  • Required parameters that are not provided in the XML configuration

Example XML service definition that would trigger this inspection: 


<service id="my_service" class="App\Service\MyService">
    <!-- Missing constructor arguments here -->
</service>

<!-- Or if constructor expects 2 arguments but only 1 is provided -->
<service id="another_service" class="App\Service\AnotherService">
    <argument type="service" id="doctrine.orm.entity_manager"/>
    <!-- Missing second argument -->
</service>
