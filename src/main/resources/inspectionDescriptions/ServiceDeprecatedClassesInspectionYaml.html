Service or class referenced in YAML configuration is deprecated

This inspection detects deprecated services and classes used in Symfony YAML service configuration. It identifies services or classes that have been marked as deprecated and should be replaced with newer alternatives.

The inspection checks for:

  • Service classes with @deprecated PHPDoc annotation
  • Classes marked with #[Deprecated] attribute
  • Services explicitly marked as deprecated in Symfony's service definition

Example YAML configuration that would trigger this inspection: 


# config/services.yaml
services:
    # This will trigger inspection if class is deprecated
    old_service:
        class: App\Legacy\OldService
        # OldService has @deprecated annotation

    # This will trigger inspection if service is deprecated
    deprecated_service:
        class: App\Service\SomeService
        # Service is marked as deprecated in Symfony

    # This will trigger inspection in dependencies
    my_service:
        class: App\Service\MyService
        arguments:
            - "@deprecated_dependency"
            # This dependency is deprecated

    # This will trigger inspection with aliases
    my_alias:
        alias: deprecated_service
        # The target service is deprecated
