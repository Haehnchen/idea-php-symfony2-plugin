Service or class referenced in XML configuration is deprecated

This inspection detects deprecated services and classes used in Symfony XML service configuration. It identifies services or classes that have been marked as deprecated and should be replaced with newer alternatives.

The inspection checks for:

  • Service classes with @deprecated PHPDoc annotation
  • Classes marked with #[Deprecated] attribute
  • Services explicitly marked as deprecated in Symfony's service definition

Example XML configuration that would trigger this inspection: 


<services>
    <!-- This will trigger inspection if class is deprecated -->
    <service id="old_service" class="App\Legacy\OldService">
        <!-- OldService has @deprecated annotation -->
    </service>

    <!-- This will trigger inspection if service is deprecated -->
    <service id="deprecated_service" class="App\Service\SomeService">
        <!-- Service is marked as deprecated in Symfony -->
    </service>

    <!-- This will trigger inspection in arguments -->
    <service id="my_service" class="App\Service\MyService">
        <argument type="service" id="deprecated_dependency" />
    </service>
</services>
